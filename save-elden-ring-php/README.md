Описание  
Проект реализует **backend + frontend** для хранения игровых сохранений (сейвов) в духе Elden Ring.  
Позволяет:
- создавать новые сейвы (имитация «отдыха у благодати»),
- просматривать список слотов и версий сохранений,
- получать данные конкретного сейва через API,
- удалять сохранения,
- автоматически делать резервные копии базы данных.

Цель — учебный проект, демонстрирующий работу с PHP, REST-подобным API, JSON, SQLite, Docker и простой игровой логикой.

Структура проекта  
- `api/` — PHP-API (`index.php`), обработка маршрутов `/api/...`.  
- `src/` — служебные модули: подключение к БД (`db.php`), логика сейвов (`EldenSave.php`), резервное копирование (`backup.php`).  
- `public/` — фронтенд (HTML/CSS/JS) в стилистике Elden Ring.  
- `tests/` — скрипт автотестов API через cURL (`test_api.php`).  
- `backups/` — резервные копии базы `elden_saves.db`.  
- `elden_saves.db` — база данных SQLite (создаётся автоматически).  
- `Dockerfile`, `docker-compose.yml` — контейнеризация и запуск.  

Запуск сайта  
Проект поднимается в Docker и открывается по адресам:

- `http://localhost:8000/public/index.html` — веб-интерфейс Elden Save Shrine  
- `http://localhost:8000/api` — health-check API

Команда для запуска из корня проекта:
docker compose up --build

Остановка:
docker compose down

Запуск тестов:
из корня проекта надо запустить php tests/test_api.php
